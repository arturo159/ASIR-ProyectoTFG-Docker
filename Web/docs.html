Guía de Portainer

Gestión visual de contenedores Docker

01

¿Por qué usamos Portainer?

Docker cuenta con su propia CLI, pero para una administración visual eficiente de contenedores, volúmenes, redes, imágenes y stacks, Portainer se ha posicionado como una de las mejores opciones. Es ameno tanto en su instalación como en su uso diario.

02

¿Qué es Portainer?

Portainer es una plataforma de gestión de contenedores con interfaz gráfica web, diseñada para administrar entornos Docker, Docker Swarm y Kubernetes sin necesidad de usar la línea de comandos para las tareas del día a día.

Convierte comandos complejos de Docker en clics simples desde el navegador, ideal tanto para administradores experimentados como para equipos que necesitan gestionar infraestructura contenerizada sin dominar toda la CLI.

03

¿Para qué usamos Portainer?

Portainer permite visualizar, crear, gestionar y monitorear todos los recursos de Docker sin necesidad de ejecutar comandos constantemente en terminal. Esto nos da ventaja a la hora de:

  • Administrar volúmenes siempre activos
  • Desplegar contenedores de manera eficiente
  • Gestionar redes y stacks
  • Visualizar logs en tiempo real
  • Controlar recursos como memoria, disco y CPU
04

Instalación Manual

Paso 1

Creación del volumen persistente

Antes de arrancar cualquier contenedor hay que asegurarse de que los datos no se pierdan. Creamos un volumen con nombre para que la configuración, usuarios y datos de Portainer persistan aunque el contenedor se borre o recree. 

docker volume create portainer_data

En este volumen vivirá la base de datos interna de Portainer: usuarios, configuraciones, entornos y credenciales.

Paso 2

Despliegue del contenedor

Con el volumen listo, lanzamos el contenedor oficial de Portainer CE:

                docker run -d \
                --name portainer \
                --restart=always \
                -p 8000:8000 \
                -p 9443:9443 \
                -v /var/run/docker.sock:/var/run/docker.sock \
                -v portainer_data:/data \
                portainer/portainer-ce:latest
Paso 3

Acceso y configuración inicial

Con el contenedor en marcha, abrimos el navegador y accedemos a:

https://localhost:9443

El navegador avisará de que el certificado no es de confianza — es lo habitual con certificados autofirmados. Aceptamos la excepción y continuamos.

La primera vez Portainer pide crear el usuario administrador. Elegimos admin como nombre y una contraseña segura. Después seleccionamos el entorno local, que aparece automáticamente al haber montado el socket de Docker, y le damos a Connect.
